Tables:

 () One table of languages, containing the ISO-639 code and the name of that
    language in that language. 

 () One table of users, containing a username and email address, as well as
    settings information indicating whether that user wants to get the
    newsletter or marketing emails. 

 () One table of users X languages (usernames and ISO-639 codes)



Creation commands (run as admin): 

create table users (id varchar(255) primary key not null, email varchar(255) not null, permission boolean not null default 0, newsletter boolean not null default 0, marketing boolean not null default 0);
create table languages (id varchar(255) primary key not null, name varchar(255) not null);
create table whospeakswhat (user varchar(255) not null references users(id), language varchar(255) not null references languages(id), level varchar(10) not null, constraint pk_whospeakswhat primary key (user, language));

"Level" in whospeakswhat table currently holds A, B, or C. In the future, we
may add additional categories, but they probably won't be more than a couple of
characters long. 




New DB:

create table users (id varchar(255) primary key not null, email varchar(255) not null, marketing boolean not null default 0, timestamp varchar(32) not null, version varchar(16) not null, loclang varchar(8) not null);

create table languages (id varchar(8) primary key not null, name varchar(32) not null);

create table whospeakswhat (user varchar(255) not null, language varchar(8) not null, speaking int not null, listening int not null, reading int not null, writing int not null, constraint fk_user foreign key (user) references users(id), constraint fk_language foreign key (language) references languages(id), constraint pk_whospeakswhat primary key (user, language), constraint chk_speaking check (speaking >= 0 and speaking <= 5), constraint chk_listening check (listening >= 0 and listening <= 5), constraint chk_reading check (reading >= 0 and reading <= 5), constraint chk_writing check (writing >= 0 and writing <= 5));
